# インターホン呼出検知 Webhook システム 仕様書

## 概要

アイホン GBM-2A（親機）の代表移報出力端子を M5StickC で監視し、呼出を検知したら Webhook を送信して外部サービスに通知するシステム。

---

## システム構成

```
玄関子機／集合玄関機
        │ 呼出
        ▼
  GBM-2A（親機）
        │ TD/TC（代表移報出力）
        ▼
    M5StickC
        │ WiFi
        ▼
   Webhook 送信先
（Make / n8n / Home Assistant 等）
```

---

## ハードウェア

### 使用機器

| 機器 | 型番 | 備考 |
|------|------|------|
| インターホン親機 | アイホン GBM-2A | AC100V駆動 |
| マイコン | M5StickC | ESP32ベース、3.3V動作 |
| 電源 | USB-C ACアダプタ | M5StickC 給電用 |

### 追加部品

| 部品 | 値 | 数量 | 用途 |
|------|----|------|------|
| 抵抗 | 10kΩ | 1 | プルアップ（TD-3.3V間） |

> **フォトカプラ（PC817）について**
> 配線が短く（数十cm）、GBM-2Aと M5StickC が近接設置のため省略。
> 両電源とも SMPS による絶縁済み出力であり、10kΩ が電流制限として機能するため、
> 直結による破損リスクは5%未満と判断。

## GBM-2A 端子仕様

### 代表移報出力（TD/TC）

| 項目 | 仕様 |
|------|------|
| 出力方式 | 無電圧オープンコレクタ（NPN） |
| 接点容量 | DC24V・5mA 以下 |
| 極性 | TD = (+) コレクタ側、TC = (-) エミッタ/GND側 |
| 出力タイミング | 呼出受信中・警報発報中 に即時出力 |

### 注意：KE 端子について

KE は `K / KS1 / KS2`（非常・コールボタン）入力回路専用のコモン端子。
**TD/TC 出力の GND としては使用不可。**

---

## 回路

### 接続図

```
GBM-2A                    M5StickC
  TD (+) ─────────────────── G36（GPIO36, 入力専用）
                               │
                             10kΩ
                               │
  TC (-) ─────────────────── GND（M5StickC GND）
                             3.3V（プルアップ電源）
```

### 動作ロジック

| 状態 | TD-TC間 | G36電位 |
|------|---------|---------|
| 待受（呼出なし） | 開放 | HIGH（3.3V、プルアップ） |
| 呼出受信中 | 導通 | LOW（0V） |

検知条件：**HIGH → LOW の立下りエッジ**

### GPIO36 の注意

- ESP32 の入力専用ピン（出力不可）
- **内部プルアップなし** → 外部 10kΩ プルアップ必須
- 3.3V 耐圧（TD の DC24V 定格内で問題なし）

## ファームウェア

### ファイル

| デバイス | ファイル |
|------|------|
| M5StickC | `m5stickc/src/main.cpp` |
| M5Atom Echo | `m5atom_echo/src/main.cpp` |

### 設定項目

| 定数 | 説明 |
|------|------|
| `SSID` | WiFi SSID（`src/config.h`） |
| `PASSWORD` | WiFi パスワード（`src/config.h`） |
| `WEBHOOK_URL` | 送信先 URL（`src/config.h`） |
| `DEBOUNCE_MS` | チャタリング除去時間（デフォルト 200ms） |
| `COOLDOWN_MS` | 再通知抑制時間（デフォルト 10,000ms = 10秒） |

### Webhook ペイロード

```json
{
  "event": "doorbell",
  "reason": "call"
}
```

HTTP POST、Content-Type: application/json

### 処理フロー

```
起動
 │
 ├─ WiFi 接続
 │
 └─ ループ
      │
      ├─ G36 読み取り（20ms周期）
      │
      ├─ HIGH→LOW エッジ検出
      │    │
      │    ├─ DEBOUNCE_MS 以内 → 無視
      │    │
      │    └─ COOLDOWN_MS 以内 → 無視
      │         │
      │         └─ Webhook POST 送信
      │              │
      │              └─ 画面表示（1秒）→ 待受表示に戻る
      │
      └─ 繰り返し
```

## 設置手順

1. GBM-2A の電源を切る
2. TD 端子に配線（10kΩ → G36）
3. TC 端子に配線（M5StickC GND）
4. `src/config.h.example` を `src/config.h` にコピーして SSID / PASSWORD / WEBHOOK_URL を設定
5. M5StickC に書き込む
6. GBM-2A の電源を入れる
7. 玄関子機から呼出テスト → Webhook 着信確認
